---
description: Processo para criar e manter a documentação do projeto
---

1. [Identificação]
   - Listar o que precisa ser documentado (API, Setup, Guia de Uso).
   - Verificar se a documentação existente está desatualizada.

2. [Redação]
   - Escrever de forma clara e concisa.
   - Usar exemplos práticos (snippets de código, comandos).
   - Seguir o padrão Markdown.

3. [Revisão]
   - Verificar a precisão técnica.
   - Garantir que os passos são reproduzíveis por terceiros.

4. [Publicação]
   - Salvar os arquivos na pasta apropriada (ex: raiz ou /docs).
   - Atualizar índices ou links se necessário.
