---
description: Processo para identificação e correção de bugs
---

1. [Reprodução]
   - Criar um caso de teste (manual ou automatizado) que reproduza o erro.
   - Documentar os passos exatos para reproduzir.

2. [Diagnóstico]
   - Analisar logs e mensagens de erro.
   - Identificar a causa raiz (Root Cause Analysis).

3. [Correção]
   - Implementar a fix.
   - Garantir que não há efeitos colaterais.

4. [Verificação]
   - Executar o caso de teste de reprodução para confirmar a correção.
   - Rodar testes de regressão se aplicável.
