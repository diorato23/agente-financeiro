---
description: Processo para realizar melhorias e refatoração no código existente
---

1. [Identificação]
   - Localizar o trecho de código a ser melhorado.
   - Documentar o estado atual e o problema (ex: performance, legibilidade, bug).

2. [Planejamento]
   - Definir a estratégia de refatoração.
   - Garantir que a mudança não quebra funcionalidades existentes (regressão).

3. [Execução]
   - Aplicar as alterações no código.
   - Manter o código limpo e seguindo os padrões do projeto.

4. [Validação]
   - Testar manualmente as funcionalidades afetadas.
   - Verificar consistência visual (se frontend) ou lógica (se backend).
