---
description: Criação estruturada de novos arquivos, componentes ou projetos
---

1. [Verificação]
   - O arquivo ou componente já existe?
   - Onde ele deve ser criado (caminho correto)?

2. [Template/Estrutura]
   - Definir o conteúdo base.
   - Seguir os padrões de nomenclatura do projeto.
   - Incluir boilerplate necessário (imports, classes, configurações).

3. [Criação]
   - Gerar o arquivo.
   - Adicionar ao controle de versão (se necessário).

4. [Integração]
   - Importar ou referenciar o novo arquivo onde for necessário.
   - Garantir que ele é reconhecido pelo sistema.
